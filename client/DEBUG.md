## Что было добавлено для отладки

### 1. В main.dart
- `debugPrint('=== APP STARTED ===')` — появляется сразу при запуске
- Исправлена индикация в роутах (был сдвинут `case '/color_picker'`)

### 2. В editor_screen.dart
- `debugPrint('=== EditorScreen initState ===')` — подтверждает загрузку редактора
- `_checkServerConnection()` — при открытии редактора проверяет доступность сервера и показывает SnackBar если сервер недоступен
- SnackBar "Отправка запроса на сервер..." — при тапе по объекту
- SnackBar "Маска готова! Проверьте и подтвердите выделение." — когда маска успешно получена
- `debugPrint` в build методе — показывает текущее состояние экрана

### 3. Визуальные индикаторы (не зависят от логов)
- Текст под FAB: "Нажмите на объект для выделения" (idle)
- Текст: "Это выделенный объект. Выберите цвет и параметры." (maskPreview)
- Кнопки "Выглядит верно" и "Выбрать заново" (maskPreview)
- Лоадер "Перекрашиваем..." (recoloring)
- SnackBar'ы с цветными фонами

## Чек-лист диагностики

1. **Запустите приложение через `run_clean.bat`** — это гарантирует что используется свежий билд
2. **В первом терминале запустите `flutter logs`**
3. **Во втором терминале запустите `flutter run`**
4. **Сделайте фото и нажмите на объект**
5. **Должны появиться:**
   - В логах: `=== APP STARTED ===`
   - В логах: `=== EditorScreen initState ===`
   - На экране: SnackBar "Отправка запроса на сервер..."
   - Через 1-3 сек: SnackBar "Маска готова!"
   - На экране: превью с синим оверлеем и кнопки выбора

## Если логов всё равно нет

### Android:
```powershell
adb devices                    # проверить что устройство подключено
adb logcat -s flutter          # смотреть логи flutter напрямую
```

### Проверить доступность сервера с телефона:
Откройте браузер на телефоне и перейдите по адресу:
```
http://212.41.29.205:8001/health
```
Должен вернуться JSON: `{"status":"healthy","device":"cuda","models_loaded":true}`

Если сервер недоступен — приложение покажет SnackBar "⚠️ Сервер недоступен"
